## M1 — Data sources

Three data sources are in scope; their roles are distinct.

### M1.1 The shipped ADRAC grid

**Source:** `legacy/Model_Rel_Candidate/DCS_Risk_DB_2025.csv` (16,295 rows, 5 columns).

**Schema:** `altitude` (ft, 18,000–40,000 in 500-ft increments), `prebreathing_time` (min, values {0, 15, 30, 45, 60}), `exercise_level` (categorical {Rest, Mild, Heavy}), `time_at_altitude` (min, 10–240 in 10-min steps), `risk_of_decompression_sickness` (target, nominally percent in [0, 100]).

**Data quality.** Two documented defects require automated repair before modeling:

1. **Scale inconsistency** — 1,221 rows (7.5% of the dataset) are mis-entered on the fraction scale `[0, 1]` instead of the documented percent scale `[0, 100]`. The target value `0.47` appears adjacent to the correctly-scaled `47`. The mis-entered rows are detected by `tinydcs.data_clean._flag_scale_rows` via a neighbour-median heuristic: a row's value $v \leq 1.0$ is flagged when its neighbour median (same exercise level; altitude, prebreathe, and time each within a small window) exceeds 1.0 and the rescaled value $100v$ is within ±30% of that neighbour median. Flagged rows are multiplied by 100. Detection is documented in `artifacts/data_quality_report.md` after running `scripts/01_clean_data.py`.

2. **Within-combo disagreements** — 26 grid cells contain two rows with distinct target values (e.g., at altitude 39,000 ft / PB 0 / Rest / 240 min the recorded values are 88 and 97). `tinydcs.data_clean._dedup_to_median` collapses each cell to its median.

After cleaning: 15,908 unique grid cells, target in percent on [0, 100].

### M1.2 Synthetic profiles from the mechanistic simulator

`scripts/02_simulate_training.py` samples exposure profiles from a reproducible distribution (seed-controlled) and evaluates the mechanistic `RutMbe1Model` on each. Each profile has:

- Altitude uniform on [18,000, 40,000] ft.
- Prebreathe duration from {0, 15, 30, 45, 60, 90, 120, 180} min with FiO₂ from {1.0: 0.8, 0.95: 0.1, 0.85: 0.1}.
- Ascent rate from {5000 fpm: 0.9, 1000 fpm: 0.1}.
- Altitude duration uniform on [10, 240] min.
- Altitude FiO₂ from {0.21: 0.8, 1.0: 0.15, 0.95: 0.05}.
- VO₂(t) trajectories at both prebreathe and altitude generated by an Ornstein–Uhlenbeck process with subject-specific mean in [0, 0.8] L·min⁻¹ whole-body, $\theta = 0.3$, $\sigma = 0.15$, bounded on [0, 1.5].

This path is currently **not** the primary training source because of the 3RUT-MBe1 reconciliation issue (§M7). It is preserved as infrastructure and as a *shape-study* tool.

## M2 — Feature engineering

A deliberately small 13-feature vector (`tinydcs.features.FEATURE_COLUMNS`) keeps the trained model below the 100 KB post-quantization target. Each feature is justified by a published mechanism.

| Feature | Mechanism / source |
|---|---|
| `altitude_ft`, `ambient_pressure_atm` | Decompression dose; ambient pressure is nonlinear in ft and ADRAC's covariate |
| `prebreathe_time_min`, `prebreathe_fio2` | Dominant protective covariate (Webb 1999) |
| `ascent_rate_fpm` | Rapid cabin depressurization vs. slow climb |
| `altitude_time_min`, `altitude_fio2` | Failure time + inspired-gas composition |
| `prebreathe_vo2_mean_lmin`, `prebreathe_vo2_peak_lmin` | Conkin 2004: exercise-enhanced prebreathe accelerates denitrogenation |
| `altitude_vo2_mean_lmin`, `altitude_vo2_peak_1min_lmin`, `altitude_vo2_integral_lmin_min` | Webb 2010, 2016: continuous VO₂ at altitude, with the Webb 1-min peak |
| `tissue_n2_ratio_360min` | Conkin-style single-compartment supersaturation ratio with water-vapor correction; serves as a physics-informed prior |

The Conkin-style `tissue_n2_ratio_360min` is computed closed-form in `tinydcs.features._tissue_n2_ratio_360` and follows:

$$
\text{TR}_{360} = \frac{P_\text{tissue N2}(t_\text{end})}{P_\text{amb}(t_\text{end})}, \quad \tau = \frac{t_{1/2}}{\ln 2}, \quad t_{1/2} = 360\ \text{min}
$$

starting from sea-level air equilibrium, with piecewise exponential updates for prebreathe → instantaneous ascent → altitude exposure.

## M3 — Surrogate model

### M3.1 Target transform

The raw target $y \in [0, 100]$ (percent after cleaning) is rescaled to $p \in [0, 1]$ and transformed on the **logit scale**:

$$
\eta = \mathrm{logit}(p) = \ln\frac{p}{1-p}, \quad p \in [\epsilon, 1-\epsilon], \ \epsilon = 10^{-6}
$$

Modelling $\eta$ instead of $p$ ensures that (a) the sigmoid inverse map keeps predictions in $(0, 1)$ without post-hoc clipping; (b) residuals are roughly homoscedastic across the 0–1 probability range.

### M3.2 Estimator

`tinydcs.surrogate.TrainConfig` uses a LightGBM regressor with MSE loss on $\eta$. Default hyperparameters: `n_estimators=400`, `learning_rate=0.05`, `num_leaves=31`, `subsample=0.9`, `colsample_bytree=0.9`, `min_data_in_leaf=20`, random state fixed. These are reasonable defaults and are not tuned against the held-out test fold.

### M3.3 Calibration — three modes supported

`tinydcs.surrogate` supports three interchangeable calibration modes, each returning a valid `TinyDcsSurrogate` bundle:

**(a) Global split-conformal** (Shafer & Vovk 2008). On the logit scale:
1. Partition the data into train (default 70%), calibration (15%), test (15%).
2. Fit LightGBM on train; predict $\hat\eta_i$ on calibration.
3. Take the conformal quantile $q = r_{(\lceil (n+1)(1-\alpha) \rceil)}$ of the absolute residuals $r_i = |\eta_i - \hat\eta_i|$, for nominal coverage $1-\alpha = 0.95$.
4. At inference emit $\hat p = \sigma(\hat\eta)$, $\hat p_\text{low} = \sigma(\hat\eta - q)$, $\hat p_\text{high} = \sigma(\hat\eta + q)$.

**(b) Mondrian (altitude-band) conformal.** Residuals are partitioned into 5,000-ft altitude bands and a separate $q_g$ is fit per band; samples with unseen bands fall back to the global quantile. This restores per-band coverage when residual scale is heteroscedastic across altitudes.

**(c) Conformalized Quantile Regression (CQR; Romano, Patterson & Candès 2019).** Two additional LightGBM *quantile* regressors are trained at the $\alpha/2$ and $1-\alpha/2$ levels on the logit scale. The nonconformity score
$$E_i = \max\big(\hat q_{\alpha/2}(x_i) - \eta_i, \ \eta_i - \hat q_{1-\alpha/2}(x_i)\big)$$
is computed on the calibration fold, and its $\lceil (n+1)(1-\alpha)\rceil$ quantile $q_\text{CQR}$ becomes the correction. The predictive interval on the logit scale is $[\hat q_{\alpha/2}(x) - q_\text{CQR},\ \hat q_{1-\alpha/2}(x) + q_\text{CQR}]$. Because the quantile regressors themselves adapt to feature-dependent spread, CQR produces asymmetric, heteroscedasticity-aware intervals. TinyDCS also supports **Mondrian-CQR**: $q_\text{CQR}$ is computed per altitude band rather than globally.

All three modes apply the final sigmoid so intervals automatically respect $[0, 1]$. LightGBM rejects monotone constraints on the quantile objective, so CQR's two quantile regressors omit monotonicity; the base mean model retains the physiological constraints.

### M3.3b Empirical calibration comparison on the ADRAC grid

| Calibration | Overall cov. | 18–23K ft | 23–28K ft | 28–33K ft | 33–38K ft | 38–43K ft | Mean width |
|---|---|---|---|---|---|---|---|
| Global | 0.869 | 0.591 | 0.933 | 0.944 | 0.967 | 0.948 | 0.29 |
| Mondrian | 0.869 | 0.583 | 0.949 | 0.945 | 0.954 | 0.955 | 0.25 |
| CQR (global q) | 0.864 | 0.589 | 0.937 | 0.945 | 0.945 | 0.937 | 0.15 |
| **Mondrian-CQR** | **0.865** | 0.589 | 0.924 | **0.959** | 0.951 | 0.937 | **0.16** |

Mondrian-CQR matches Mondrian's per-band coverage outside the lowest band while producing intervals roughly half as wide. The 18,000–23,000 ft shortfall is invariant across all three methods because it is driven by **target distribution pathology, not residual variance**: ~40% of rows in that band have target exactly zero, and neither a mean regressor's residuals nor a quantile regressor's output adequately captures the spread of the remaining non-zero rows when trained on the whole grid.

### M3.3c Two-stage zero-inflated calibration (*implemented*)

The principled fix for the 18,000–23,000 ft shortfall is a **two-stage zero-inflated mixture** in the spirit of Lambert (1992):

1. A binary LightGBM classifier $P(y = 0 \mid x)$ is trained on the full training fold with `is_zero = (y ≤ 10⁻⁶)` as the target.
2. A continuous LightGBM regressor of $\eta = \text{logit}(y)$ is trained only on rows with $y > 0$, inheriting the physiological monotonicity constraints of the mean surrogate.
3. A split-conformal quantile is computed on the non-zero calibration residuals of the continuous regressor.
4. At inference, a gate threshold (default 0.5) on $P(y=0|x)$ routes each sample to either a zero-anchored interval $[0, \text{zero\_upper\_bound}]$ (default upper = 0.02) or the continuous branch's conformal interval. The mixture point estimate is $(1 - P(y=0|x)) \cdot \sigma(\hat\eta)$.

This is implemented in `tinydcs.surrogate.ZeroInflatedCalibration` and selectable via `train_surrogate(use_zero_inflated=True)` or the `--zi` flag on `scripts/04_train_adrac_surrogate.py`. Empirically it lifts the 18,000–23,000 ft band from 0.583 under every conformal-only method to **0.964**, and overall coverage from 0.869 to **0.960**, with a slight MAE improvement to 0.020 (from 0.022 under Mondrian conformal). This is the **default** calibration for Paper-1 headline numbers.

### M3.4 Out-of-distribution abstention

`tinydcs.surrogate.OODDetector` fits a Mahalanobis distance in feature space:

$$
d(\mathbf{x}) = \sqrt{(\mathbf{x} - \boldsymbol{\mu})^\top \Sigma^{-1} (\mathbf{x} - \boldsymbol{\mu})}
$$

with a 0.1% Ledoit–Wolf-style shrinkage for numerical stability. The abstention threshold is the empirical 99th percentile of training distances, giving an expected 1% in-distribution false-abstention rate. The detector does **not** make probabilistic abstention decisions; it emits a boolean flag alongside every prediction.

## M4 — Evaluation

The `scripts/03_train_surrogate.py` evaluation script produces a single JSON blob with:

| Category | Metrics |
|---|---|
| **Point accuracy** | MAE, RMSE, R² |
| **Discrimination** | Brier score; AUROC thresholded at 10% risk |
| **Calibration** | Van Calster 2019 slope + intercept (weighted logistic recalibration); reliability bins (10 equally-spaced bins) |
| **Fidelity** | Bland–Altman bias + 95% limits of agreement |
| **Uncertainty** | Empirical conformal coverage, average interval width |
| **Edge efficiency (planned)** | Inference latency, model size, RAM footprint |

Pre-specified acceptance thresholds (for Paper 1 submission):

- MAE on held-out test ≤ 0.03 (absolute probability units);
- Calibration slope ∈ [0.9, 1.1], intercept magnitude < 0.05;
- Empirical conformal coverage ≥ 94% at nominal 95%;
- OOD detector abstains ≥ 90% on deliberately-out-of-envelope inputs;
- Post-quantization model size ≤ 100 KB, latency ≤ 1 ms on Cortex-M4.

## M5 — Validation strategy

Three held-out splits are pre-specified:

1. **Random split** (default) — uniform shuffling, reported first.
2. **Leave-one-altitude-out** (planned) — stratify by altitude band in 5,000-ft steps and hold out one band at a time. Detects extrapolation failures at the envelope edges.
3. **Leave-one-profile-family-out** (planned) — hold out all rows matching a specific (altitude, prebreathe, exercise) signature to test generalization to unseen profile shapes.

Paper 1 will report random split as the primary comparison with prior work (Han 2023) and leave-one-altitude-out as the robustness analysis.

## M6 — Reproducibility

- **Dependencies** pinned in `requirements.txt` and mirrored in `pyproject.toml`.
- **Random seeds** fixed at every stage (`--seed 42` in the simulation CLI; `random_state=42` in `TrainConfig`).
- **Data cleaner** is deterministic and idempotent; the quality report records every transformation.
- **Artifacts** (cleaned parquet, training parquet, surrogate joblib, metrics JSON, figures) are written to `artifacts/` (git-ignored) and reproducible from raw inputs + script invocations.
- **Test suite** (14 tests, `pytest tests/ -q`) covers the data cleaner, simulator, features, and end-to-end surrogate round-trip.

## M7 — 3RUT-MBe1 reconciliation

### The observation

Running the vendored `mechanistic/rut_mbe1.py` on Gerth's own ADRAC-validation profiles (NEDU TR 18-01 Fig. 16) produces P(DCS) values 4–5 orders of magnitude smaller than published:

| Profile | This implementation | Published (approx.) |
|---|---|---|
| A: 90 min PB, 35K ft, light ex, 180 min | 1.2e-5 | ~20–30% |
| B: 30 min PB, 25K ft, heavy ex, 240 min | 2.2e-5 | ~40–60% |
| C: 15 min PB, 22.5K ft, heavy ex, 240 min | 2.3e-5 | ~50–70% |
| D: 0 PB, 18K ft, heavy ex, 360 min | 3.7e-5 | ~30–50% |
| E: 75 min PB, 30K ft, rest, 240 min | 1.8e-5 | ~10–20% |

### Suspect parameters

Appendix C of NEDU TR 18-01 defines several scaled parameters whose defaults in `mechanistic/rut_mbe1.py` (Table 3 of the same report) may not match Gerth's fitted values:

- **Λ (`lambda_cm_inv` = 100.0)** — scale factor for the dimensionless formulation; the report notes Λ can be chosen arbitrarily but the other scaled parameters must be consistent.
- **`n0_b_total_nuclei` (1.198)** — total initial nuclei count; enters the hazard exponentially via Eq. (C.8).
- **`gain_g_hazard` (6.188e-2)** — direct multiplier on the integrated hazard.
- **`sigma_surface_tension_dyne_per_cm` (30.0)** and **`sigma_c_factor` (19.64)** — control bubble nucleation thresholds.
- **Numerical damping** in the recursion across thousands of 0.5-min steps may also contribute.

### Reconciliation plan

A scalar "fit-the-output-to-Figure-16" patch is explicitly rejected as scientifically invalid — it would conceal the underlying bug rather than locate it, and a Table-3 parameter error will resurface under any profile outside the five validation points Gerth published. The reconciliation is therefore structured as a **line-by-line audit** against NEDU TR 18-01, not a curve fit.

Until that audit completes, 3RUT-MBe1 is **not** a training target. It is used for monotonicity/shape studies only. The ADRAC surrogate remains Paper 1's primary deliverable.

### Appendix-C audit checklist (open task for an agent with NEDU TR 18-01 in hand)

The checklist below is sized so that someone with access to a scanned copy of NEDU TR 18-01 (Gerth, "Three Region Unsteady-state Tissue Multibubble estimator 1", 2018, DTIC AD1101527) can work through it mechanically, equation-by-equation, without guessing.

The goal for each item is: **(a) name the equation, (b) identify which Table-3 constant(s) enter, (c) compute the reference value the report prints in Appendix D, (d) compare to our implementation.** Any mismatch > 1 % on a stepwise quantity is almost certainly the source of the 4–5 OoM output discrepancy.

- [ ] **C.1 — Ambient-pressure scaling.** Verify `altitude_ft_to_mmhg()` matches the U.S. Standard Atmosphere table Gerth references. Our current ADRAC module uses `(1 − 6.87535e-6·h)^5.2559 · 760`. Confirm this is what Gerth's Table 2 altitudes convert to.
- [ ] **C.2 — Tissue N₂ washout time constants.** Confirm that the four-compartment half-times used internally match Table 3 values (not Workman-Bühlmann generic values). Dump the compartmental N₂ tensions at the end of a 90-min prebreathe and check against Appendix D, if present.
- [ ] **C.3 → C.7 — Supersaturation $P_{ss}(t)$.** Run one validation profile (Profile A: 90 min PB → 35 kft → 180 min light ex) and log $P_{ss}$ at 1-min intervals. The report prints $P_{ss}$ values for anchor time points; confirm ours match to at least three sig figs.
- [ ] **C.8 — Bubble number density $n_b$ evolution.** This is the equation that uses `n0_b_total_nuclei` (1.198 in our Table-3 copy). Critical: confirm whether 1.198 is a *count* (integer-adjacent) or a *scaled density* in the dimensionless formulation. A unit error here alone could produce many orders of magnitude in the hazard exponent.
- [ ] **C.9 — Bubble radius recursion $\hat{r}_{k+1} = f(\hat{r}_k, P_{ss,k}, \Lambda, \sigma)$.** This uses Λ (`lambda_cm_inv`), $\sigma$ (`sigma_surface_tension_dyne_per_cm`), and $\sigma_c$ (`sigma_c_factor`). Compute $\hat{r}$ for Profile A at t = 5, 30, 60, 120, 180 min and compare to Appendix D tables if present. The ratio of our $\hat{r}$ to the reference must be 1.00 ± 0.01 at every step; otherwise every downstream recursion rolls forward the error.
- [ ] **C.10 → C.20 — Integrated hazard $H(t) = \int g \cdot n_b(\tau) \cdot V_b(\tau) \, d\tau$.** This is the equation where `gain_g_hazard` (6.188e-2) enters. Confirm the dimensions. The report specifies $g$ has units such that $g \cdot n_b \cdot V_b$ has units of min⁻¹. Check by inspection that our numerical $g$ produces that result. A dimensionless-vs-dimensional mix-up in $g$ is the single most plausible source of a 10⁴–10⁵ error.
- [ ] **C.21 — Final P(DCS) from integrated hazard.** $P(\text{DCS}) = 1 - \exp(-H(T_{\text{end}}))$. Confirm our code does not accidentally use $H(T)$ in dimensional cm·s⁻¹ units when Gerth's is dimensionless.
- [ ] **Numerical step size.** The report uses a specific `dt`. Our recursion uses `0.5 min` by default. Verify that Gerth's published timestep matches and that stability at `dt = 0.5 min` has been demonstrated for altitude transitions (ascent segments can be < 1 min in real profiles but much longer in validation profiles — the schedule matters).
- [ ] **Ascent segment handling.** Some 3RUT-MBe1 formulations treat the ascent as a continuous ramp; others as an instantaneous step. Confirm which Gerth used for Fig. 16 and match our behaviour in `mechanistic/rut_mbe1.py::_simulate_profile()`.
- [ ] **Λ normalization.** Λ = 100.0 cm⁻¹ in our defaults. The report states Λ can be chosen arbitrarily *provided the other scaled parameters are recomputed consistently*. Confirm by dimensional argument that the Table-3 values for $n^0_b$, $g$, $\sigma_c$ in our code are the ones Gerth published *for* Λ = 100 cm⁻¹, and not, for example, the Λ = 1 cm⁻¹ normalisation. A Λ-mismatch between defaults and Table-3 values is exactly the kind of silent error that produces order-of-magnitude offsets.

When the audit concludes, write a one-page addendum in this section listing:

1. Which equation held the bug.
2. The corrected parameter or code.
3. Re-run of the five Profile A–E validation points showing the corrected output ± 10 % of Fig. 16 anchors.

Only after that can 3RUT-MBe1 be promoted to (a) training target for a second-generation surrogate, or (b) reference signal alongside ADRAC in a multi-target ensemble.

## M8 — Deviations from the ideal methods

In the interest of transparent reporting:

- Paper 1's primary target is **ADRAC**, not 3RUT-MBe1. The earlier project framing called for a 3RUT-MBe1 surrogate; M7 explains the pivot.
- Individual variability (Paper 2 scope) is **not** represented. The surrogate predicts population-average risk.
- External validation (Paper 3 scope) is **not** present. The held-out "test" set is random within the same synthetic grid distribution.
- Multimodal wearable fusion (HR, HRV, SpO₂, skin temp) is **not** present in v0.x. It enters in Paper 2.
- The INT8 ONNX edge export is planned for Paper 1 but not yet implemented (see `scripts/04_export_onnx.py` placeholder in the roadmap).

Each of these is tracked in `CHANGELOG.md` under "Known issues / open questions" and in `docs/publication-roadmap.md` as explicit future work with a scope.
